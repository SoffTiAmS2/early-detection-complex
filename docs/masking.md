# Masking

## Что маскировать

- реальные имена хостов;
- реальные IP-адреса;
- реальные учетные данные;
- названия внутренних серверов;
- прямые пути к административным интерфейсам.

## Как делать в проекте

- использовать имена `sensor1`, `sensor2`, `sensor3` в учебной документации;
- реальные адреса заменять лабораторной подсетью из `inventory/network.yml`;
- секреты хранить не в репозитории, а в локальных `.env`;
- в логах не использовать реальные учетные данные предприятия.

## Почему это важно

Honeypot должен имитировать инфраструктуру, но не раскрывать ее настоящие параметры. Для ВКР достаточно логической модели сети.
