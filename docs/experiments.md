# Experiments

## Что измерять

- время развертывания сенсора;
- факт фиксации TCP-подключения;
- доставку события в центральный узел;
- работу collector API;
- восстановление после перезапуска;
- корректность отображения статуса.

## Минимальный эксперимент

1. Запустить центральный узел.
2. Запустить `sensor1`.
3. Подключиться к Cowrie SSH/Telnet порту.
4. Проверить событие в `GET /api/events`.
5. Изменить host-port в `config/project.json` и переустановить сенсор через `scripts/deploy_sensor.sh`.

## Пример тестового подключения

```sh
nc <sensor-ip> 2222
```

Ожидаемый результат:

- Cowrie запишет событие в `logs/cowrie.json`;
- log-agent отправит событие на центральный узел;
- collector API покажет новое событие.
